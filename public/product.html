Cloudflare Wrangler Automation

Automate Wrangler deployments, D1 database management, and configuration — Clodo Framework integrates deeply with the Wrangler CLI to provide validated, repeatable deployments across environments. Core components include WranglerDeployer, WranglerD1Manager, and WranglerConfigManager.

Minimal programmatic example

// Wrangler deploy + D1 migrations
import { WranglerDeployer } from '@tamyla/clodo-framework/deploy';
const d = new WranglerDeployer();
await d.deploy('production');
await d.runD1Migrations('production');

Key capabilities

  • Automated discovery and validation of wrangler.toml for multiple environments
  • Safe Wrangler command execution (retries, timeouts, parsed output)
  • D1 database lifecycle management (create, migrate, list, validate)
  • Interactive and programmatic deployments: node bin/deployment/enterprise-deploy.js deploy --interactive

Quick example

// Programmatic deployment using DeploymentOrchestrator
import { DeploymentOrchestrator } from '@tamyla/clodo-framework/deployment';

const orchestrator = new DeploymentOrchestrator({ environment: 'production' });
await orchestrator.deploy('production');

Troubleshooting & best practices

  • Verify authentication with wrangler whoami; Clodo runs the same check via validateWranglerSetup().
  • Use WranglerConfigManager to inspect/fix wrangler.toml (checks name, main, compatibility_date).
  • If D1 issues occur, inspect migrations: wrangler d1 migrations list, and apply with wrangler d1 migrations apply <database>.
