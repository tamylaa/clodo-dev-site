Implementation Examples

Custom Logging

export default {
  async fetch(request, env, ctx) {
    const startTime = Date.now();
    
    // Log request details
    console.log({
      method: request.method,
      url: request.url,
      ip: request.headers.get('cf-connecting-ip'),
      timestamp: new Date().toISOString()
    });
    
    const response = await handleRequest(request, env);
    
    // Log response details
    const duration = Date.now() - startTime;
    console.log({
      status: response.status,
      duration,
      cached: response.headers.get('cf-cache-status')
    });
    
    return response;
  }
}

Performance Tracking

// Send metrics to analytics service
async function trackMetric(env, name, value, tags = {}) {
  await env.ANALYTICS.put(
    `metric:${name}}:${Date.now()}}`,
    JSON.stringify({
      value,
      tags,
      timestamp: Date.now()
    })
  );
}

// Usage in handler
export default {
  async fetch(request, env) {
    const start = Date.now();
    
    const response = await fetch('https://api.example.com');
    const duration = Date.now() - start;
    
    await trackMetric(env, 'api_latency', duration, {
      endpoint: 'example.com',
      status: response.status
    });
    
    return response;
  }
}

Error Tracking

async function reportError(env, error, context = {}) {
  const errorData = {
    message: error.message,
    stack: error.stack,
    context,
    timestamp: new Date().toISOString(),
    url: context.url || 'unknown'
  };
  
  // Log to console (visible in wrangler tail)
  console.error('ERROR:', errorData);
  
  // Send to error tracking service
  await fetch('https://errors.example.com/api/errors', {
    method: 'POST',
    body: JSON.stringify(errorData)
  });
}

// Usage
try {
  // Your code
} catch (error) {
  await reportError(env, error, {
    userId: request.headers.get('user-id'),
    endpoint: request.url
  });
  return new Response('Error occurred', {status: 500});
}

User Event Tracking

// Track user events
async function trackEvent(env, eventName, eventData) {
  const event = {
    name: eventName,
    data: eventData,
    timestamp: Date.now(),
    sessionId: crypto.randomUUID()
  };
  
  // Store in KV for later analysis
  await env.EVENTS.put(
    `event:${Date.now()}}:${eventName}}`,
    JSON.stringify(event)
  );
  
  // Batch send to analytics
  if (shouldFlushEvents()) {
    await flushEventsToAnalytics(env);
  }
}

// Usage
export default {
  async fetch(request, env) {
    const method = request.method;
    
    if (method === 'POST') {
      const data = await request.json();
      await trackEvent(env, 'form_submission', {
        formId: data.formId,
        duration: data.duration
      });
    }
    
    return new Response('ok');
  }
}

Monitoring Best Practices

üéØ Establish Baselines

Before optimizing, measure your current performance. Know what "normal" looks like.

  • Average response time
  • Error rate
  • P99 latency
  • Cache hit rate

‚ö†Ô∏è Set Alerts

Don't wait for users to report issues. Set up proactive alerts for anomalies.

  • Error rate > 1%
  • Latency P95 > 200ms
  • Cache hit rate < 80%
  • Availability < 99.9%

üîÑ Track Trends Over Time

Compare performance week-over-week and month-over-month to spot gradual degradation.

  • Performance by time of day
  • Seasonal patterns
  • Long-term trends
  • Correlation with deployments

üö® Debug Systematically

Use structured logging and tracing to quickly locate issues.

  • Add correlation IDs to requests
  • Log at key decision points
  • Include timing information
  • Capture error context

üìà Measure Business Metrics

Don't just track technical metrics. Monitor what matters to your business.

  • Conversion rate
  • User retention
  • Feature adoption
  • Revenue per user

üíæ Retain Data Appropriately

Balance detailed data retention with cost. Keep different data for different periods.

  • 1 day: Full resolution data
  • 30 days: Hourly aggregates
  • 1 year: Daily aggregates
  • Archive as needed for compliance
