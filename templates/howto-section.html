How to deploy a Clodo service to the edge (quick start)

  1. Install Clodo Framework: Install the package and CLI helpers with npm install @tamyla/clodo-framework.
  2. Scaffold a service: Run npx create-clodo-service my-edge-service --type data-service to generate a Workers-ready starter project.
  3. Configure bindings: Add D1 bindings, environment variables and wrangler.toml settings for routes, accounts and secrets.
  4. Test & benchmark: Run locally (npm run dev or wrangler dev) and measure latency improvements against a cloud-hosted variant.
  5. Deploy & monitor: Deploy to Cloudflare, run synthetic latency tests, and use monitoring/logs to verify performance and reliability.

Tip: Start tiny, benchmark often, and only move critical latency-sensitive logic to the edge.
