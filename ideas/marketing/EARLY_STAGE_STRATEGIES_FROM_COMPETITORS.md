### 1. **Next.js (2016 - Unknown â†’ Dominant)**

**When They Started:**
- Unknown framework
- React was popular, but SSR was hard
- Competing with Create React App, Gatsby

**Early-Stage Strategies:**

#### A. **"Show the Pain, Solve It Visually"**
- Created comparison: "Here's React SSR (complex) vs Next.js (simple)"
- Visual demos showing code reduction
- "Before/After" code comparisons

**Tactic for Clodo:**
- Show: "Manual service creation (100 hours) vs Clodo (1 hour)"
- Visual: Side-by-side code comparison
- Demo: Time-lapse of creating 50 services

#### B. **"Built by Vercel, Used by Vercel"**
- Vercel (the company) used Next.js for their own site
- "Dogfooding" = credibility
- Showed it worked at scale

**Tactic for Clodo:**
- Use Clodo to build Clodo's own infrastructure
- Show: "We built X services with Clodo"
- Case study: "How we use Clodo internally"

#### C. **"Free Templates & Examples"**
- Created tons of examples
- GitHub repos with working code
- "Learn by example" approach

**Tactic for Clodo:**
- 20+ service templates on GitHub
- Example apps (full working projects)
- "Copy-paste ready" code

#### D. **"Conference Talks"**
- Guillermo (creator) spoke at conferences
- Not selling, just educating
- "Here's how we solved SSR"

**Tactic for Clodo:**
- Speak at Cloudflare events
- Local meetups
- Online conferences
- Topic: "Automating Microservices on Workers"

#### E. **"Documentation as Marketing"**
- Incredible documentation
- Tutorials, guides, examples
- SEO goldmine

**Tactic for Clodo:**
- Best-in-class docs
- Step-by-step tutorials
- Video guides
- SEO-optimized content

**Key Lesson:** Next.js didn't market itself as "better React" - it showed how it solved a specific pain (SSR complexity).
